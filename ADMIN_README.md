# 靈學管理後台

## 概述

為 LMS 項目創建的管理後台前端頁面，包含完整的用戶管理、課程管理和系統設置功能。

## 已創建文件

### 1. admin-login.html - 登錄頁
- **功能**：管理員登錄入口
- **特點**：
  - 簡潔的居中卡片式設計
  - 紫色主題配色 (#7C3AED)
  - 支持密碼顯示/隱藏切換
  - 錯誤提示（不使用 alert）
  - 支持 Enter 鍵提交
  - 響應式設計
  - 深色模式支持

### 2. admin.html - 管理後台主頁
- **功能**：單頁應用（SPA），包含多個管理面板
- **結構**：
  - **左側邊欄**（固定，深紫色 #4C1D95）：
    - Logo：靈學管理後台
    - 導航項：儀表板、課程管理、用戶管理、系統設置
    - 底部：當前用戶信息、退出登錄
  - **頂部欄**：
    - 搜索框
    - 通知圖標
    - 用戶頭像下拉菜單
  - **主內容區**：
    - Panel 1: 儀表板 - KPI 卡片 + 最近課程
    - Panel 2: 課程管理 - 課程列表 + 篩選 + 分頁 + CRUD 操作
    - Panel 3: 用戶管理 - 用戶列表 + 角色管理
    - Panel 4: 系統設置 - 修改密碼

### 3. js/admin.js - 管理後台 JavaScript
- **功能**：所有前端邏輯和 API 交互
- **核心模塊**：
  - JWT Token 管理
  - API 請求封裝（統一處理 401）
  - 面板切換邏輯
  - 加載狀態管理
  - Toast 通知系統
  - 模態框管理（自定義，不使用 confirm）
  - 課程 CRUD 操作
  - 用戶管理操作
  - 分頁邏輯
  - 表格排序
  - 搜索和篩選

### 4. css/admin.css - 管理後台樣式
- **功能**：管理後台專用樣式
- **內容**：
  - 響應式佈局
  - 動畫效果（fadeIn, slideIn, rotate）
  - 模態框樣式
  - Toast 通知樣式
  - 表格樣式（包含排序指示器）
  - 分頁樣式
  - 深色模式優化
  - 側邊欄過渡效果
  - 自定義滾動條

## 技術規範

### API 交互
- **基地址**: `http://localhost:3001/api`
- **認證方式**: JWT Bearer Token
- **401 處理**: 自動清除 token 並跳轉登錄頁

### 表單驗證
- 所有表單字段都有前端驗證
- 密碼最小長度：6 位
- 密碼確認需要匹配
- 必填字段檢查

### 用戶體驗
- 加載狀態顯示（Spinner）
- Toast 通知（成功/錯誤/信息）
- 刪除操作確認（自定義模態框，不使用 confirm）
- 表格排序
- 分頁（每頁 10 條）
- 搜索和篩選

### 響應式設計
- 桌面端（≥1024px）：側邊欄固定顯示
- 平板端（768px-1023px）：側邊欄可折疊
- 移動端（<768px）：側邊欄隱藏，通過漢堡菜單打開

### 深色模式
- 支持深色/淺色模式切換
- 所有組件都有深色模式樣式
- 使用 Tailwind CSS 的 `dark:` 前綴

## 使用說明

### 登錄流程
1. 訪問 `admin-login.html`
2. 輸入用戶名和密碼
3. 點擊登錄或按 Enter 鍵
4. 登錄成功後 JWT 保存到 localStorage
5. 自動跳轉到 `admin.html`

### 儀表板
- 顯示 4 個 KPI 卡片（課程總數、學員總數、平均評分、總收入）
- 顯示最近添加的 5 個課程

### 課程管理
- 搜索課程（按標題）
- 篩選課程（按分類、難度）
- 表格排序（點擊表頭）
- 分頁瀏覽
- 新建課程
- 編輯課程
- 刪除課程（需確認）
- 查看章節（展開/折疊）

### 用戶管理
- 用戶列表（支持分頁）
- 修改用戶角色（下拉選擇）
- 刪除用戶（需確認）

### 系統設置
- 修改密碼
  - 輸入當前密碼
  - 輸入新密碼（至少 6 位）
  - 確認新密碼

### 退出登錄
- 點擊側邊欄底部的「退出登錄」
- 或點擊頭像下拉菜單中的「退出登錄」
- 需確認後才執行

## API 接口說明

### 認證相關
- `POST /api/auth/login` - 登錄
- `POST /api/auth/change-password` - 修改密碼

### 儀表板
- `GET /api/stats/dashboard` - 獲取儀表板統計數據

### 課程管理
- `GET /api/courses` - 獲取課程列表（支持分頁、搜索、篩選、排序）
- `GET /api/courses/:id` - 獲取單個課程詳情
- `POST /api/courses` - 創建課程
- `PUT /api/courses/:id` - 更新課程
- `DELETE /api/courses/:id` - 刪除課程
- `GET /api/courses/:id/chapters` - 獲取課程章節

### 用戶管理
- `GET /api/users` - 獲取用戶列表（支持分頁）
- `PUT /api/users/:id` - 更新用戶（如修改角色）
- `DELETE /api/users/:id` - 刪除用戶

## 瀏覽器兼容性

- 現代瀏覽器（Chrome, Firefox, Safari, Edge）
- 需要 JavaScript 啟用
- 支持本地存儲（localStorage）

## 安全性

- 所有 API 請求都通過 JWT 認證
- 401 未授權自動清除 token
- 表單數據前後端雙重驗證
- 刪除操作需要二次確認

## 注意事項

1. **API 服務器**: 需要後端 API 服務器運行在 `http://localhost:3001`
2. **JWT 存儲**: Token 保存在 localStorage
3. **跨域問題**: 如遇 CORS 錯誤，需要在後端配置 CORS
4. **圖片路徑**: 課程封面圖片 URL 需要確保可訪問
5. **初始數據**: 首次使用需要通過 API 創建管理員賬戶
